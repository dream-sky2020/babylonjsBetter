# Shared Core

这个目录用于放置 **Web(Vite)** 与 **Tauri** 可复用的核心能力。

## 目标

- 统一战斗逻辑，避免双端逻辑漂移。
- 统一 Babylon 场景创建逻辑，减少重复维护成本。
- 将平台相关代码（浏览器/Tauri API）隔离到适配层。

## 边界约束

- 本目录下代码应尽量保持“平台无关”。
- 不直接依赖 `window`、`document`、`@tauri-apps/api`。
- 如果需要平台能力，请通过接口注入（Port/Adapter 模式）。

## 计划放入的内容

- `battle/`: 战斗状态机、结算规则、技能计算
- `scene/`: Babylon 场景工厂与通用系统
- `types/`: 共享类型定义
- `ports/`: 跨平台能力接口（存档、日志、事件等）

## 使用方式（建议）

- Vite 入口注入 Web 适配器后调用共享核心。
- Tauri 入口注入 Tauri 适配器后调用共享核心。
- UI 组件只负责显示和交互，不承载核心规则。

## 共享数据目录

- `shared/data/` 用于双端共享的静态数据。
- 在前端通过 `/shared/data/...` 读取（开发与构建后保持一致）。
- 该目录默认按“只读资源”使用；运行时可写数据请使用平台存储能力。
