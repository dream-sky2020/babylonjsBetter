# 桌宠模块说明（Tauri + Three.js）

本文档说明当前项目中的桌宠骨架模块，包含前端渲染、Tauri 窗口、前后端通信以及后续扩展建议。

## 1. 模块目标

- 在桌面左下角显示一个透明悬浮窗作为桌宠。
- 使用 Three.js 进行轻量渲染（正交相机 + 单精灵）。
- 通过 Tauri `invoke` + `event` 进行前后端通信。

## 2. 目录与文件

### 前端（Web）

- `desktop-pet.html`  
  桌宠页面入口（Vite 多页面之一）。
- `desktopPetMain.tsx`  
  React 挂载入口。
- `desktopPet/DesktopPetApp.tsx`  
  桌宠前端核心逻辑（渲染、动画、通信、调试 UI）。
- `desktopPet/desktop-pet.css`  
  桌宠样式与 debug 面板样式。

### 后端（Tauri / Rust）

- `src-tauri/src/main.rs`  
  Tauri 启动、窗口定位、命令处理、事件广播。
- `src-tauri/tauri.conf.json`  
  窗口配置（透明、置顶、尺寸、任务栏、devUrl）。
- `src-tauri/icons/*`  
  应用图标资源（Windows 需要 `icon.ico`）。

## 3. 运行方式

```bash
npm install
npm run tauri:dev
```

如果出现 `cargo ... program not found`，说明 Rust/Cargo 环境未安装或未生效，需要先安装 Rust 并重开终端。

## 4. 通信设计

### 命令（前端 -> 后端）

- `pet_get_state`：读取当前桌宠状态
- `pet_set_state`：设置桌宠状态

### 事件（后端 -> 前端）

- `pet://state_changed`：桌宠状态变化广播
- `pet://heartbeat`：心跳事件（用于调试和健康检查）

## 5. 当前状态结构（示例）

```ts
type PetStatePayload = {
  mood: 'idle' | 'happy' | 'sleep';
  animation_speed: number;
};
```

后续建议加上：

- `version`：协议版本
- `request_id`：请求追踪
- `timestamp`：统一时序

## 6. 常见修改入口

- 改桌宠动画/显示：`desktopPet/DesktopPetApp.tsx`
- 改窗口行为（透明、置顶、位置）：`src-tauri/tauri.conf.json` + `src-tauri/src/main.rs`
- 改前后端协议：`desktopPet/DesktopPetApp.tsx` + `src-tauri/src/main.rs`

## 7. 后续扩展建议

- 支持 sprite sheet 切帧动画（idle/walk/sleep）
- 增加鼠标拖拽移动窗口
- 增加点击穿透切换
- 增加配置持久化（窗口位置、当前动作、是否置顶）
- 增加统一日志通道（`pet://log` / `pet://error`）
